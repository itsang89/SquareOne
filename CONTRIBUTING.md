# Contributing to SquareOne

First off, thank you for considering contributing to SquareOne! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct:
- Use welcoming and inclusive language.
- Be respectful of differing viewpoints and experiences.
- Gracefully accept constructive criticism.
- Focus on what is best for the community.
- Show empathy towards other community members.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issue Tracker.
- Describe the bug in detail, including steps to reproduce.
- Mention your OS and browser version.

### Suggesting Enhancements
- Open an issue with the "enhancement" label.
- Explain why this feature would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code follows the existing style (use Prettier/ESLint).

## Development Process

1. **Setup**: Follow the "Getting Started" guide in the `README.md`.
2. **Branching**: Use descriptive branch names (e.g., `feat/add-dark-mode` or `fix/login-error`).
3. **Commits**: Use conventional commit messages (e.g., `feat: ...`, `fix: ...`, `docs: ...`).
4. **Testing**: Run `npm run dev` and verify your changes manually.

## Style Guide

- Use TypeScript for all new files.
- Follow the Neo-brutalist design patterns established in `components/NeoComponents.tsx`.
- Use functional components and hooks.
- Keep components small and focused.

## Questions?

Feel free to open an issue for any questions or clarifications!
